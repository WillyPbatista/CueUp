# 06.02 - Crear partida desde sala ready

## Objetivo

Cuando los dos jugadores de una sala esten listos, crear una partida persistida y mover a ambos clientes hacia la ruta de juego.

## Conceptos que aprenderas

- Transiciones de estado entre `rooms` y `matches`.
- Validaciones antes de crear una partida.
- Realtime para navegar a ambos clientes.
- Idempotencia para evitar crear dos partidas.

## Pasos

1. Crear servicio `startMatchFromRoom(roomId, hostId)`.
2. Validar que la sala tenga exactamente 2 jugadores.
3. Validar que ambos jugadores tengan `ready = true`.
4. Insertar una fila en `matches`.
5. Cambiar `rooms.status` a `playing`.
6. Suscribir `RoomPage` a cambios de `rooms.status`.
7. Navegar a `/game/:matchId` cuando exista partida.
8. Mostrar errores si la sala no esta lista.

## Resultado esperado

Al abrir dos navegadores en la misma sala, cuando ambos marcan ready se puede iniciar la partida y ambos clientes navegan al juego sin refrescar.

## Checklist

- [ ] Se crea una fila en `matches`.
- [ ] No se crean partidas duplicadas.
- [ ] Solo se inicia si hay 2 jugadores.
- [ ] Solo se inicia si ambos estan ready.
- [ ] La sala pasa a `playing`.
- [ ] Ambos navegadores navegan a `/game/:matchId`.
- [ ] Se documenta la diferencia entre estado de sala y estado de partida.

## Nota de aprendizaje

Anotar por que una transicion multiplayer debe ser idempotente y validada cerca de la base de datos.
