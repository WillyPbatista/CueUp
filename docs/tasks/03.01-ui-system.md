# 03.01 - Construir el sistema visual

## Objetivo

Definir la identidad visual de CueUp y crear componentes consistentes para la app.

## Conceptos que aprenderas

- Design tokens.
- Tailwind.
- Variantes de componentes.
- Responsive design.
- Estados visuales.

## Pasos

1. Revisar `src/styles/theme.css`.
2. Definir colores base de mesa, acento, oro y fondo.
3. Crear variantes para Button y Badge.
4. Crear estados hover, focus y disabled.
5. Crear layout principal para paginas.

## Resultado esperado

CueUp empieza a tener identidad visual propia y componentes reutilizables.

## Checklist

- [ ] Tokens visuales revisados.
- [ ] Componentes usan colores del tema.
- [ ] Botones tienen variantes.
- [ ] Inputs tienen estados claros.
- [ ] Layout responde en mobile y desktop.

## Nota de aprendizaje

Anotar que es un design token y por que evita estilos repetidos.

## Commit sugerido

```txt
feat(ui): define visual system
```
