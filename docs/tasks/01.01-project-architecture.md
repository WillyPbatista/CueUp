# 01.01 - Definir arquitectura del proyecto

## Objetivo

Entender y documentar como se organiza CueUp antes de seguir creando funcionalidades.

## Conceptos que aprenderas

- Arquitectura feature-first.
- Separacion de responsabilidades.
- Diferencia entre UI, dominio, servicios y estado.
- Como planear una aplicacion antes de programarla.

## Pasos

1. Revisar `docs/architecture.md`.
2. Comparar la estructura objetivo con la estructura actual.
3. Crear carpetas faltantes solo cuando una tarea las necesite.
4. Mantener la arquitectura simple hasta que el juego pida mas complejidad.

## Resultado esperado

La estructura base del proyecto esta clara y documentada.
