# 04.01 - Ordenar arquitectura de aplicacion

## Objetivo

Separar correctamente paginas, features, servicios, stores y tipos.

## Conceptos que aprenderas

- Feature-first.
- Providers.
- Servicios.
- Stores.
- Tipos compartidos.

## Pasos

1. Revisar `docs/architecture.md`.
2. Crear convenciones para cada carpeta.
3. Mover codigo de UI compartida a `components/`.
4. Crear features cuando exista logica de producto.
5. Evitar meter toda la logica en `App.tsx`.

## Resultado esperado

El codigo nuevo tiene un lugar claro y el proyecto no crece de forma desordenada.

## Checklist

- [ ] `App.tsx` solo coordina providers/router.
- [ ] `pages/` contiene pantallas.
- [ ] `features/` contiene logica de producto.
- [ ] `services/` contiene acceso externo.
- [ ] `stores/` contiene estado cliente.
- [ ] `types/` contiene tipos compartidos.

## Nota de aprendizaje

Escribir una regla simple para decidir donde va un archivo nuevo.

## Commit sugerido

```txt
refactor(app): organize application layers
```
