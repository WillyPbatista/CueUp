# 01.03 - Configurar herramientas de calidad

## Objetivo

Preparar un entorno profesional con linting, formato y hooks de git.

## Conceptos que aprenderas

- ESLint.
- Prettier.
- Husky.
- lint-staged.
- Diferencia entre errores de codigo y formato.

## Pasos

1. Revisar `eslint.config.js`.
2. Revisar `.prettierrc`.
3. Revisar `.prettierignore`.
4. Revisar la configuracion `lint-staged` en `package.json`.
5. Ejecutar `npm run quality`.

## Resultado esperado

El proyecto puede validar calidad con un solo comando.

## Checklist

- [x] ESLint configurado.
- [x] Prettier configurado.
- [x] Husky instalado.
- [x] lint-staged configurado.
- [x] Script `quality` disponible.

## Nota de aprendizaje

Anotar que problema resuelve cada herramienta.

## Commit sugerido

```txt
chore(quality): configure linting and formatting
```
