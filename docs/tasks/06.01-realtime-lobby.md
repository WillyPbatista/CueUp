# 06.01 - Crear lobby realtime

## Objetivo

Permitir que dos jugadores creen, vean y se unan a salas sincronizadas en tiempo real.

## Conceptos que aprenderas

- Supabase Realtime.
- Presence.
- Broadcast.
- CRUD.
- Estados de sala.

## Pasos

1. Crear pagina Lobby.
2. Listar salas `waiting`.
3. Crear sala nueva.
4. Unirse a una sala por codigo.
5. Sincronizar cambios con Realtime.
6. Mostrar jugadores conectados.

## Resultado esperado

Al abrir dos navegadores, ambos ven cambios de salas sin refrescar manualmente.

## Checklist

- [x] Se pueden crear salas.
- [x] Se pueden listar salas.
- [x] Se puede entrar a una sala.
- [x] Dos clientes ven cambios realtime.
- [x] Se muestra estado de conexion.
- [x] Se limita la sala a 2 jugadores.
- [x] El ready status cambia en tiempo real en `RoomPage`.

## Nota de aprendizaje

Anotar que diferencia hay entre leer de base de datos y suscribirse a cambios.

Extra: `getRoomPlayerReadyStatus(roomId)` lee el estado actual, mientras que `subscribeToRoomPlayers(roomId)` avisa cuando cambia `room_players`. La UI necesita ambos: snapshot inicial y canal realtime.
