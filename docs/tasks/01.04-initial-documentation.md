# 01.04 - Crear documentacion inicial

## Objetivo

Crear una base de documentacion que acompanara el desarrollo del juego.

## Conceptos que aprenderas

- README.
- Roadmap.
- Documentacion de arquitectura.
- Convencion de commits.
- Bitacora de aprendizaje.

## Pasos

1. Revisar `README.md`.
2. Revisar `docs/architecture.md`.
3. Revisar `docs/roadmap.md`.
4. Revisar `docs/commits.md`.
5. Revisar `docs/learning-notes.md`.
6. Mantener `docs/tasks/` como guia de avance.

## Resultado esperado

El proyecto explica que es CueUp, como se ejecuta, que stack usa y hacia donde va.

## Checklist

- [x] README actualizado.
- [x] Arquitectura documentada.
- [x] Roadmap actualizado.
- [x] Convenciones de commits documentadas.
- [x] Learning notes listas.
- [x] Tareas guiadas creadas.

## Nota de aprendizaje

Anotar por que documentar temprano ayuda a no avanzar a ciegas.

## Commit sugerido

```txt
docs(project): add initial project documentation
```
